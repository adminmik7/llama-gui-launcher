# Llama.cpp Model Launcher

GUI-приложение на tkinter для запуска llama-server с настраиваемыми параметрами.

## Возможности

- **Выбор файлов**: путь к `llama-server.exe`, GGUF-модели и mmproj-файла
- **Настройки сервера**: хост (автоопределение локального IP), порт (по умолчанию 1414), размер контекста, GPU-слои (-1 или >= 1), CPU-потоки (по умолчанию `os.cpu_count()`), batch/ubatch size (по умолчанию 512)
- **Параметры генерации**: temperature, top-k, top-p, parallel, KV-cache K/V (f16, q8_0, q4_0, q4_1, q5_0, q5_1, turbo3, turbo4), CPU MoE, reasoning budget
- **Дополнительные флаги**: Flash Attention, Continual Batching, Jinja шаблон, No MMAP, KV Unified, Preserve Thinking
- **Логирование**: вывод stdout сервера с таймстампами, копирование выделения (Ctrl+C), контекстное меню
- **Конфигурация**: сохранение и загрузка настроек в JSON

## Запуск

```bash
python llama_launcher.pyw
```

## Структура

- `llama_launcher.pyw` — основной файл приложения (скрывает консоль на Windows)

## История изменений

- **Раздельные KV-cache**: единый параметр `--cache-type-k/--cache-type-v` заменён на два отдельных поля `KV-cache K` и `KV-cache V`
- **Авто-определение хоста**: поле Host теперь автоматически подставляет локальный IP машины
- **Порт по умолчанию**: изменён с пустого на `1414`
- **GPU слои**: валидация разрешает только `-1` или значения `>= 1`
- **Автодетект llama-server**: удалён, путь к серверу теперь задаётся вручную
