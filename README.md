# llama-gui-launcher

GUI-лаунчер для запуска `llama-server` из [llama.cpp](https://github.com/ggerganov/llama.cpp) с графическим интерфейсом на Tkinter. Позволяет удобно выбирать модели, настраивать параметры сервера и генерации, сохранять/загружать конфигурации и отслеживать логи в реальном времени.

## Изменения

### Недавние изменения
- Увеличен размер шрифта в панели логов (9 → 11)
- Убрано дублирующее сообщение "Сервер завершился" из лога (остается только код выхода)

## Возможности

- Выбор пути к `llama-server.exe` и GGUF-модели через диалог файлов
- Поддержка визуализации через `mmproj`-файлы (multimodal)
- Кастомные chat-шаблоны (`.jinja`)
- Настройка параметров сервера: хост, порт, контекст, GPU-слои, потоки, batch size
- Настройка параметров генерации: temperature, top-k, top-p, parallel
- KV-cache quantization (K/V): f16, bf16, f32, q8_0, q4_0, q4_1, iq4_nl, q5_0, q5_1
- Дополнительные опции: Flash Attention, Continual Batching, Jinja, No MMAP, KV Unified, Preserve Thinking
- MoE (Mixture of Experts) и reasoning budget
- Валидация всех числовых полей с диапазонами
- Сохранение и загрузка конфигураций в JSON
- Автосохранение последнего пути к конфиге
- Мониторинг логов сервера в реальном времени с копированием
- Валидация параметров перед запуском
- Безопасное закрытие: остановка сервера и предложение сохранить конфиг



## Требования

- **Python 3.8+** (только стандартная библиотека, дополнительные зависимости не нужны)
- **`llama-server.exe`** — бинарник из [llama.cpp](https://github.com/ggerganov/llama.cpp) (сборка с CUDA/ROCm/Metal по желанию)
- **Windows** — интерфейс оптимизирован под Windows (Vista/Clam темы Tkinter)
- **GGUF-модель** — файл модели в формате GGUF
- **ОЗУ/VRAM** — зависит от размера модели и контекста

## Установка

1. **Скачайте `llama-server.exe`**
   - Склонируйте [llama.cpp](https://github.com/ggerganov/llama.cpp)
   - Соберите с нужным бэкендом (CUDA, ROCm, Metal, CPU) или скачайте готовую сборку
   - Положите `llama-server.exe` в папку с лаунчером или укажите путь в GUI

2. **Скопируйте файлы лаунчера**
   - `llama_launcher.pyw` — основной файл (`.pyw` запускает без окна консоли)
   - `qwen_template.jinja` — пример chat-шаблона для Qwen

3. **Запустите**
   - Двойной клик по `llama_launcher.pyw`
   - Или: `python llama_launcher.pyw`

## Использование

### Шаг 1: Выбор файлов

| Поле | Описание | Пример |
|------|----------|--------|
| Путь к llama-server.exe | Бинарник llama.cpp | `C:\models\llama-server.exe` |
| Файл модели (.gguf) | GGUF-модель для загрузки | `C:\models\llama-3.1-8b-Q4_K_M.gguf` |
| Файл mmproj (визуализация) | файл для поддержки vision (опционально) | `C:\models\mmproj-model.gguf` |
| Файл chat template | Jinja-шаблон для чата (опционально) | `C:\models\qwen_template.jinja` |

### Шаг 2: Настройка сервера

| Параметр | По умолчанию | Диапазон | Описание |
|----------|-------------|----------|----------|
| Хост | автоопределение | любой IP | Адрес для прослушивания |
| Порт | `1414` | 1–65535 | Порт HTTP API |
| Контекст | `120000` | 1–1 000 000 | Размер контекстного окна (`-c`) |
| GPU слои | `999` | -1 или ≥1 | Количество слоёв на GPU (`-ngl`), `-1` = все |
| Потоки CPU | авто (os.cpu_count) | 1–1024 | Количество потоков (`-t`) |
| Batch size | `512` | 1–4096 | Размер батча (`-b`) |
| UBatch size | `512` | 1–4096 | Размер убатча (`-ub`) |

Каждый параметр (кроме хоста и порта) можно отключить чекбоксом.

### Шаг 3: Настройка генерации

| Параметр | По умолчанию | Диапазон | Описание |
|----------|-------------|----------|----------|
| Temperature | `0.6` | 0.01–2.0 | Случайность генерации (`--temp`) |
| Top-k | `20` | 1–500 | K лучших токенов (`--top-k`) |
| Top-p | `0.95` | 0.0–1.0 | Nucleus sampling (`--top-p`) |
| Parallel | `2` | 1–32 | Параллельные запросы (`--parallel`) |
| KV-cache K | `q4_0` | f16/bf16/f32/q8_0/q4_0/q4_1/iq4_nl/q5_0/q5_1 | Квантование KV-кэша K |
| KV-cache V | `q4_0` | f16/bf16/f32/q8_0/q4_0/q4_1/iq4_nl/q5_0/q5_1 | Квантование KV-кэша V |
| CPU MoE | `0` | 0–1000 | Число экспертов на CPU (`--n-cpu-moe`) |
| Reasoning | `0` | 0–100000 | Reasoning budget (`--reasoning-budget`) |

### Шаг 4: Дополнительные опции

| Опция | По умолчанию | Описание |
|-------|-------------|----------|
| Flash Attention | ✅ включена | `-fa on` — ускорение внимания |
| Continual Batching | ✅ включена | `--cont-batching` — непрерывная обработка |
| Jinja шаблон | ✅ включена | `--jinja` — использование Jinja для шаблонов |
| No MMAP (Windows) | ✅ включена | `--no-mmap` — отключение mmap (рекомендуется на Windows) |
| KV Unified | ✅ включена | `--kv-unified` — объединённый KV-кэш |
| Preserve Thinking | ✅ включена | Сохраняет `<thinking>`-блоки в ответе |

### Шаг 5: Запуск

1. Нажмите **▶ Запустить**
2. Отслеживайте логи в панели «Лог сервера»
3. Для остановки нажмите **⏹ Остановить**

При закрытии окна лаунчер предложит сохранить конфиг и остановит сервер.

## Чек-лист оптимизации

### Для GPU с большим VRAM (24GB+)
- GPU слои: `999` (все слои на GPU)
- Context: `120000` (максимум, если хватает памяти)
- KV-cache: `f16` для максимального качества
- Flash Attention: включена
- Batch size: `512`–`1024`

### Для GPU с ограниченным VRAM (8–12GB)
- GPU слои: подберите экспериментально (начните с `20`–`30`)
- Context: `8192`–`32768`
- KV-cache: `q4_0` или `q8_0`
- Batch size: `256`–`512`

### Для CPU
- GPU слои: `0` или `-1` (все на CPU)
- Потоки CPU: число физических ядер
- Context: `8192`–`32768` (зависит от ОЗУ)
- No MMAP: включена (Windows)
- Batch size: `512`–`1024`

### Для быстрого ответа
- Temperature: `0.7`–`1.0`
- Top-k: `40`–`50`
- Top-p: `0.9`–`0.95`
- Parallel: `1`–`2`

### Для креативности
- Temperature: `0.8`–`1.2`
- Top-k: `50`–`100`
- Top-p: `0.95`–`0.99`

## FAQ

**Сервер не запускается.**
- Проверьте путь к `llama-server.exe` — файл должен существовать
- Убедитесь, что порт не занят другим процессом
- Проверьте логи — llama.cpp выводит причины ошибок

**Модель не загружается.**
- Убедитесь, что файл в формате GGUF
- Проверьте, что хватает VRAM/ОЗУ для выбранного контекста
- Попробуйте уменьшить контекст или GPU слои

**Медленная генерация.**
- Увеличьте GPU слои (перенесите больше слоёв на GPU)
- Уменьшите context size
- Включите Flash Attention
- Увеличьте batch size (если хватает VRAM)

**Что такое mmproj?**
- Файл `mmproj` — это модель проекции для мультимодальных моделей (vison + текст). Нужен для моделей типа LLaVA, Qwen2-VL и других, которые работают с изображениями.

**Как переключить на другой чат-шаблон?**
- Выберите файл `.jinja` в поле «Файл chat template»
- В проекте есть `qwen_template.jinja` — шаблон для Qwen3

**Где сохраняется конфиг?**
- При сохранении вы выбираете путь вручную
- Последний путь к конфигу сохраняется в `.last_config_path` и загружается автоматически при старте

**Можно ли использовать на Linux/macOS?**
- Интерфейс Tkinter работает кроссплатформенно, но `creationflags=subprocess.CREATE_NO_WINDOW` оптимизирован под Windows. На Linux/macOS уберите эту опцию в коде.
